**Keep only tasks whose correct answer reduces to a single numeric value, a single MC letter, or an exact string-match label; normalize units, currency, percents, MC formats, fractions, and sentence-embedded numbers; and filter out multi-value answers, symbolic-form answers, ambiguous text labels, and unsupported scientific notation.**

# Final Specification for Ground-Truth Filtering and Normalization

## 1. **Filter Out Entirely**

Exclude these samples from RL training.

### 1.1 Multi-value answers

* Lists, sets, tuples, unordered or ordered multi-element answers
  Examples:

  * `{2,-3}`, `(1,2,3)`, `[radio, television, newspaper]`, `"2, 3, 5"`

### 1.2 Symbolic expressions without a single numeric target

* Expressions where multiple algebraic forms are valid and cannot be canonically reduced

  * `2(x+3)`
  * `\sqrt{4}`
  * Trig identities, complicated polynomial rearrangements

### 1.3 Scientific notation with positive exponents

* `"3.2e5"` (unless you normalize it yourself first)

### 1.4 Ambiguous textual answers

* Answers where the correct label cannot be reliably determined via exact or case-insensitive string match

  * Example: explanations, long sentences, multi-word descriptions with synonyms
  * Anything requiring semantic matching instead of exact or very-close string matching

---

## 2. **Keep, but Normalize (Reformat) Before Training**

These answers are allowed but must be canonicalized.

### 2.1 Multiple Choice → **canonical uppercase letter**

Normalize all MC ground truth to a single letter:

* `"A red car"` → `A`
* `"A) red car"` → `A`
* `"(B) blue house"` → `B`
* `"Choice (C): green tree"` → `C`
* `"I choose the blue house."` → `C` *only if mapping is known*

Rule:
**Ground truth for MC must be a single letter `A`–`Z` after normalization.**

### 2.2 Percent → decimal

* `"45%"` → `0.45`
* `"31.74%"` → `0.3174`

### 2.3 Units → number

Strip units completely:

* `"80 cm^2"` → `80`
* `"57.6 cubic inches per second"` → `57.6`
* `"80 square centimeters"` → `80`

### 2.4 Currency → number

Remove currency symbols, commas, spaces:

* `"$1,234.50"` → `1234.5`
* `"€0.75"` → `0.75`

### 2.5 Sentence answers → extract the numeric answer

* `"The answer is 42."` → `42`
* `"Area is 57.6 square units"` → `57.6`

### 2.6 Arithmetic, fractions, LaTeX fractions → reduce to a numeric value

* `"1/2 + 1/3"` → `5/6`
* `"\frac{1}{2}"` → `0.5`
* `"3^2 + 4^2"` → `25`

### 2.7 Scientific notation (negative exponent only)

* `"1.2e-3"` → `0.0012`

---

## 3. **Keep As-Is (No Reformat Needed)**

These ground truths are safe without modification.

### 3.1 Single numeric values

* `42`, `3.14`, `0.75`, `0.5000`
* `"1,000,000"` (optionally normalize)

### 3.2 Expressions that canonically represent a single number

* `"1 + 2 + 3"`
* `"2*(3+4)"`
* `"x^2 + 2x + 1"` (if this exact form is the expected canonical answer)

### 3.3 LaTeX fractions

* `\(\frac{1}{2}\)`
  Evaluates to a single number.

### 3.4 **Exact or near-exact string-match labels (non-numeric)**

Allowed **only if** correctness can be judged by literal string equality (case-insensitive) after trimming.

Examples:

* `"True"` vs `"TRUE"`
* `"cat"` vs `"Cat"`
* `"red car"` vs `"Red Car"`

Not allowed:

* Synonyms, paraphrases, long descriptions
* Cases needing semantic comparison
* Cases where multiple wordings could be “equivalent” but do not match exactly

Rule:
**Accept nonnumeric labels only when exact string equivalence is the correctness criterion.**

# JSON Output Instructions (Per Sample)

Each sample should be converted into a JSON object:

```
{
  "keep": true/false,
  "reason": "<string>",
  "normalized_ground_truth": "<string or null>",
  "type": "<numeric | mc | string | null>"
}
```

Fields:

* `"keep"`: whether this sample is suitable for RL training
* `"reason"`: concise explanation of keeping or filtering
* `"normalized_ground_truth"`: canonical answer if keep=true; otherwise null
* `"type"`: `"numeric"`, `"mc"`, or `"string"` when keep=true; otherwise null

---

# Decision Rules

Apply these rules to every sample, using both the **question** and the **ground-truth answer**.

---

## 1. Filter Out Entirely (keep=false)

Reject samples whose ground-truth answer fits any of these categories:

### 1.1 Multi-value answers

Lists, tuples, sets, or any answer containing multiple values.

### 1.2 Non-canonical symbolic expressions

Symbolic expressions not guaranteed to reduce to a single numeric value.

### 1.3 Ambiguous textual labels

Free-text answers whose correctness requires semantic interpretation (synonyms, paraphrases, multi-word descriptions not guaranteed to match exactly).

### 1.4 Unsupported scientific notation

Positive exponents such as `"3.2e5"` unless pre-normalized.

If filtered:

```
"keep": false,
"normalized_ground_truth": null,
"type": null
```

---

## 2. Normalize If Keepable (keep=true)

If not filtered out, normalize:

### 2.1 Multiple Choice → single uppercase letter

Normalize any MC-style ground truth to `"A"`, `"B"`, `"C"`, etc.

### 2.2 Numeric ground truths

Normalize using:

* Unit stripping
* Percent conversion → decimal
* Currency stripping
* Thousand-separator cleanup
* Sentence extraction
* Fraction/expression evaluation
* Negative scientific-notation conversion

### 2.3 String-match labels

Allowed **only** when correctness is literal (case-insensitive) string match.
